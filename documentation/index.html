FloatKit
v2.0 Docs

A lightweight floating social sidebar widget. Drop two files into any website and get animated social buttons — no frameworks, no jQuery, no build tools.

Zero Dependencies 6 Themes 6 Positions 21 Networks ~5kb minified IE11+
💡

Use the Live Configurator on the demo page to visually pick your settings, then copy the generated code directly into your project.

Installation

FloatKit ships as two files. Copy them into your project — that's all you need.

1

Download the package

After purchase on CodeCanyon, unzip the archive. You'll find floatkit.css and floatkit.js in the /dist folder.

2

Copy files to your project

Place both files anywhere in your project, e.g. /assets/floatkit/. No npm, no webpack, no build step required.

3

Link in your HTML

Add the CSS in <head> and the JS before </body>. Then call FloatKit.init().

Quick Start

Minimum setup — get running in under 2 minutes.

html 
<!-- In <head> -->
<link rel="stylesheet" href="floatkit.min.css">

<!-- In <body> — widget container -->
<div id="floatkit-widget"></div>

<!-- Before </body> -->
<script src="floatkit.min.js"></script>
<script>
FloatKit.init({
  position: 'left',
  theme:    'solid',
  networks: [
    { id: 'facebook',  url: 'https://facebook.com/yourpage' },
    { id: 'instagram', url: 'https://instagram.com/yourhandle' },
    { id: 'twitter',   url: 'https://x.com/yourhandle' }
  ]
});
</script>
✓

That's it. The widget will appear on the left side of the page and auto-adapts to mobile screens.

All Options

Pass any of these to FloatKit.init({}). All fields are optional except networks.

OptionTypeDefaultDescription
containerstring'#floatkit-widget'CSS selector for the mount point
positionstring'left'Widget position. See Positions section
themestring'solid'Visual theme. See Themes section
shapestring'rounded'Button shape: rounded | circle | sharp | square
sizestring'md'Button size: sm | md | lg | xl
blurbooleanfalseEnable out-of-focus blur effect on hover
showLabelsbooleantrueShow tooltip labels on hover
hideMobilebooleanfalseHide widget on screens < 768px
mobileMoveBottombooleantrueAuto-move to bottom bar on mobile
scrollHidebooleanfalseHide widget when user is near top of page
scrollHideOffsetnumber100px from top to trigger scroll-hide
networksarray—Array of network objects (required)

Network object fields

FieldTypeRequiredDescription
idstringYesNetwork identifier (see Networks list)
urlstringYesFull URL to your profile or page
labelstringNoCustom tooltip text (overrides default)
countstringNoBadge text, e.g. '2.1k' or '1,234'

Themes

Set the theme option to one of the values below.

ValueDescription
solidFull brand colors per network. Best for colorful, vibrant sidebars
glassFrosted glass look with backdrop blur. Great on image backgrounds
outlineTransparent with colored border per network. Clean and minimal
glowSolid colors with neon glow shadow. Ideal for dark pages
minimalDark semi-transparent buttons. Works on any background
lightWhite buttons with subtle shadow. Best for light/white backgrounds
💡

Use the glass theme when you have a full-page background image. Use light theme for white or very light page backgrounds.

Blur Effect

When blur: true, hovering over one button causes all others to blur and fade out — keeping visual focus on the active button. Inspired by Socialsider's out-of-focus feature.

javascript
FloatKit.init({
  blur: true,   // ← enable blur effect
  theme: 'solid',
  networks: [...]
});

You can also toggle it dynamically after init using the JS API: widget.setBlur(true)

Count Badges

Each network can display a small red badge with a follower count or custom text.

javascript
networks: [
  {
    id:    'facebook',
    url:   'https://facebook.com/yourpage',
    count: '12k'    // ← shows badge with this text
  },
  {
    id:    'instagram',
    url:   'https://instagram.com/handle'
    // no count = no badge
  }
]

Mobile Behavior

FloatKit has two mobile options:

OptionBehavior
mobileMoveBottom: trueWidget moves to a centered bottom bar on screens < 768px (default)
hideMobile: trueWidget is completely hidden on mobile screens
⚠️

If both hideMobile and mobileMoveBottom are true, hideMobile wins — the widget will be hidden.

JavaScript API

FloatKit.init() returns an API object you can use to update the widget after initialization.

javascript
// Save the returned instance
const widget = FloatKit.init({ ... });

// Change theme on the fly
widget.setTheme('glass');

// Move to different position
widget.setPosition('right');

// Change button shape
widget.setShape('circle');

// Change size
widget.setSize('lg');

// Toggle blur effect
widget.setBlur(true);

// Toggle hover labels
widget.setLabels(false);

// Swap network buttons dynamically
widget.rebuild([
  { id: 'twitter', url: '#' },
  { id: 'discord', url: '#' }
]);

// Remove widget from DOM
widget.destroy();

WordPress Integration

To add FloatKit to a WordPress site, add the following to your theme's functions.php:

php — functions.php
add_action( 'wp_enqueue_scripts', function() {
    wp_enqueue_style(
        'floatkit',
        get_template_directory_uri() . '/assets/floatkit.min.css'
    );
    wp_enqueue_script(
        'floatkit',
        get_template_directory_uri() . '/assets/floatkit.min.js',
        [], '2.0', true
    );
} );

add_action( 'wp_footer', function() {
    echo '<div id="floatkit-widget"></div>
    <script>
    FloatKit.init({
        position: "left",
        theme: "solid",
        blur: false,
        networks: [
            { id: "facebook",  url: "https://facebook.com/yourpage" },
            { id: "instagram", url: "https://instagram.com/yourhandle" },
            { id: "twitter",   url: "https://x.com/yourhandle" }
        ]
    });
    </script>';
} );
💡

Copy floatkit.min.css and floatkit.min.js to your theme's /assets/ folder first.

FAQ

Does it work without jQuery?

Yes. FloatKit is pure Vanilla JavaScript — no jQuery, no external libraries, no dependencies whatsoever.

Can I use multiple widgets on one page?

Yes. Give each container a unique id and call FloatKit.init() separately for each, passing the matching container selector.

Can I add a custom network not in the list?

Currently the widget supports the 21 built-in networks. Custom SVG icon support is planned for v2.1 via a iconSvg field in the network object.

The blur effect looks different on Safari

The blur uses CSS filter: blur() which is fully supported in all modern browsers. Safari may render it slightly differently but it functions correctly.

How do I change button URLs after init?

Use widget.rebuild(networks) — pass a new array of network objects. The widget will re-render with the new configuration.

Can I use this with React / Vue?

Yes. Call FloatKit.init() inside a useEffect (React) or mounted (Vue) hook, and call widget.destroy() on component unmount to clean up.
